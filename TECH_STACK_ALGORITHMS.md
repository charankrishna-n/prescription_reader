## 🔧 Algorithms Used

### 1. **JWT Authentication Algorithm**
```
Algorithm: HS256 (HMAC with SHA-256)
Process:
  1. User login with email/password
  2. Verify credentials against stored data
  3. Generate JWT token with user info
  4. Return token to client
  5. Client stores token in localStorage
  6. Axios interceptor adds token to all requests
  7. Backend verifies token signature
  8. Extract user info from token payload
```

### 2. **Alarm Time Checking Algorithm**
```
Algorithm: Real-time Clock Comparison
Process:
  1. Set interval to check every 1000ms (1 second)
  2. Get current system time
  3. Format as HH:MM (24-hour format)
  4. Compare with each alarm's alarm_time
  5. If match found:
     a. Check if alarm already triggered (activeAlarms state)
     b. If not triggered, trigger alarm
     c. Mark as triggered for 60 seconds
     d. Play audio alert
     e. Show browser notification
     f. Display visual indicator
  6. After 60 seconds, reset trigger state
```

### 3. **Time Until Alarm Calculation**
```
Algorithm: Countdown Timer
Process:
  1. Parse alarm time (HH:MM)
  2. Create alarm date object for today
  3. If alarm time < current time:
     - Set alarm for tomorrow
  4. Calculate difference in milliseconds
  5. Convert to hours and minutes
  6. Format as "Xh Ym"
  7. Update every second
```

### 4. **Image Capture Algorithm**
```
Algorithm: Canvas-based Photo Capture
Process:
  1. Access camera via getUserMedia API
  2. Stream video to <video> element
  3. On capture button click:
     a. Get canvas context
     b. Set canvas dimensions = video dimensions
     c. Mirror image: context.scale(-1, 1)
     d. Draw video frame to canvas
     e. Convert canvas to blob
     f. Create File object from blob
     g. Convert to base64 for preview
     h. Stop camera stream
```

### 5. **Medicine Extraction Algorithm**
```
Algorithm: AI-Powered OCR with Confidence Scoring
Process:
  1. Send prescription image to Gemini API
  2. API analyzes image using vision model
  3. Extract structured data:
     - Medicine name (primary text)
     - Dosage (quantity + unit)
     - Frequency (times per day/week)
     - Duration (days/weeks/months)
     - Meal context (before/after/with food)
  4. Calculate confidence score (0-1)
  5. Return JSON with all fields
  6. Display in UI with confidence percentage
```

### 6. **State Management Algorithm**
```
Algorithm: Context-based Global State
Process:
  1. AuthContext manages:
     - User login state
     - User type (patient/doctor)
     - JWT token
     - Logout functionality
  
  2. DataContext manages:
     - Medicines array
     - Calendar items
     - Tracker items
     - Notifications
     - Prescriptions
     - Adherence data
  
  3. Callback functions with dependencies:
     - useCallback prevents unnecessary re-renders
     - Dependencies array tracks state changes
     - Memoization optimizes performance
```

### 7. **Notification Permission Algorithm**
```
Algorithm: Browser Notification Request
Process:
  1. Check if Notification API available
  2. Check current permission status
  3. If permission not granted:
     - Request permission from user
     - Wait for user response
  4. If granted:
     - Create notification object
     - Set title, body, icon
     - Display to user
  5. Handle permission denied gracefully
```

### 8. **Audio Alert Generation Algorithm**
```
Algorithm: Web Audio API Synthesis
Process:
  1. Create AudioContext
  2. Create Oscillator node
  3. Create GainNode for volume control
  4. Connect: Oscillator → GainNode → Destination
  5. Set frequency: 800 Hz (sine wave)
  6. Set initial gain: 0.3 (30% volume)
  7. Exponential ramp down over 2 seconds
  8. Start oscillator
  9. Stop after 2 seconds
  10. Clean up audio context
```

### 9. **Data Persistence Algorithm**
```
Algorithm: localStorage-based Persistence
Process:
  1. User profile data:
     - Serialize to JSON
     - Store in localStorage with key 'userProfile'
     - Retrieve on app load
     - Parse JSON back to object
  
  2. JWT token:
     - Store in localStorage on login
     - Retrieve for API requests
     - Clear on logout
  
  3. Prescription images:
     - Store as base64 strings
     - Persist across tab navigation
     - Available in DataContext
```

### 10. **Calendar Date Calculation Algorithm**
```
Algorithm: Month Navigation and Date Mapping
Process:
  1. Get current month/year
  2. Calculate days in month
  3. Calculate first day of month (0-6)
  4. Create array with:
     - Empty slots for days before month starts
     - Day numbers (1-31)
  5. Display in 7-column grid (Sun-Sat)
  6. For each day, filter medicines by date
  7. Display medicine badges on calendar
  8. Handle month navigation (prev/next)
```

### 11. **Adherence Calculation Algorithm**
```
Algorithm: Medicine Adherence Percentage
Process:
  1. Get all tracker items for user
  2. Count total medicines prescribed
  3. Count medicines marked as taken
  4. Calculate: (taken / total) * 100
  5. Return adherence percentage
  6. Display with visual indicator
```

### 12. **Form Validation Algorithm**
```
Algorithm: Input Validation
Process:
  1. Check required fields not empty
  2. Validate email format (regex)
  3. Validate password length (min 6 chars)
  4. Validate number ranges (age 0-150)
  5. Validate time format (HH:MM)
  6. Show error messages
  7. Disable submit button if invalid
  8. Clear errors on successful submission
```

## 📊 Data Flow

### Authentication Flow
```
User Input (email, password)
    ↓
LoginPage Component
    ↓
API Call: POST /api/auth/login
    ↓
Backend Verification
    ↓
JWT Token Generation
    ↓
Store in localStorage
    ↓
Update AuthContext
    ↓
Redirect to Dashboard
```

### Prescription Upload Flow
```
User selects image (file or camera)
    ↓
Preview displayed
    ↓
User clicks "Analyze Prescription"
    ↓
API Call: POST /predict (multipart/form-data)
    ↓
Backend: Send to Gemini API
    ↓
Gemini: Extract medicines
    ↓
Return structured JSON
    ↓
Display medicines in UI
    ↓
User adds to Tracker/Calendar/Notifications
    ↓
Data stored in backend
    ↓
Update DataContext
```

### Alarm Trigger Flow
```
User sets alarm (time + medicine)
    ↓
Store in notifications array
    ↓
Every 1 second: Check current time
    ↓
If time matches alarm time:
    ├── Play audio (Web Audio API)
    ├── Show browser notification
    ├── Display visual indicator
    └── Mark as triggered for 60s
    ↓
After 60s: Reset trigger state
```

## 🔐 Security Technologies

### 1. **JWT Authentication**
- Algorithm: HS256
- Secret key stored on backend
- Token includes: user_id, email, user_type
- Expiration: Configurable (default 24 hours)
- Verification on every protected endpoint

### 2. **CORS (Cross-Origin Resource Sharing)**
```python
CORSMiddleware(
    allow_origins=["http://localhost:5173"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"]
)
```

### 3. **Password Security**
- Stored in-memory (demo only)
- In production: Use bcrypt hashing
- Minimum length validation
- Never transmitted in plain text

### 4. **HTTPS (Production)**
- SSL/TLS encryption
- Secure cookie transmission
- Protected token storage

### 5. **Input Validation**
- Pydantic models validate all inputs
- Type checking
- Range validation
- Format validation

### 6. **Error Handling**
- Generic error messages (no sensitive info)
- Detailed logging on backend
- Graceful error recovery on frontend
